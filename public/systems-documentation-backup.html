What is this system?

The CI eLearner is a highly interactive, state-driven React Single Page Application (SPA). It replaces legacy static slide decks with an immersive, "neo-brutalist meets glassmorphism" learning experience.

It parses complex regulatory requirements (Conditions of Participation, Homebound Status) into a 3-stage interactive flow: Foundation ‚Üí Expanded Context ‚Üí Validation Challenge.

Core Philosophy

Maximize learner retention through high-contrast UI, immediate validation feedback, and frictionless performance.

Architecture
Client-Side SPA

Application State vs Persistence

Data Architecture

The application operates entirely in-memory using React's virtual DOM. State changes (like advancing cards or answering quizzes) are instantaneous and require zero server round-trips.

If deployed as a standalone web app (Vercel/Netlify), session data is transient. If deployed within an LMS wrapper (SCORM), terminal states are passed to the LMS API via JavaScript bridges.

Performance Win: By decoupling UI state from database constraints, interactions run at 60fps without loading spinners.

Design System & Styling Engine

Tailwind JIT Configuration

The app uses Tailwind's utility classes with arbitrary values for exact brand-kit matching without bloating the config file.

  • Brand Palette

Advanced Visual Patterns

Complex effects using CSS transforms and variables maintain high performance.

  • Dynamic Mouse Tracking Glow

    A global mousemove listener updates CSS variables (--mouse-x, --mouse-y) for a cursor-following radial gradient without React re-renders.

  • Glassmorphism & Depth

    Extensive use of backdrop-blur-xl and translucent backgrounds (bg-white/85) creates physical depth against dark underlays.

The component architecture is monolithic by design, contained within App.tsx to ensure portability. It imports a singular icon library to keep bundle sizes minimal.

Runtime Sequence: The Tri-Panel Flow

How data flows from initial load to challenge validation.

1

Initial Render (Main Mode)

Mount

App mounts. cardIndex is 0. panelMode is 'main'. Virtual DOM renders the Learning Objective and Key Points grid.

const card = cards[cardIndex];
// Renders main block
{panelMode === 'main' && ( ... )}
2

Internal Navigation

Update

User clicks "Initialize Audio" or "Enter Challenge". The panelMode state updates, causing an immediate re-render with CSS transition animations.

setPanelMode('additional');
setStatusText('System: Playing Audio Protocol');
// key prop forces remount ‚Üí .animate-glitch CSS
3

Challenge Validation

Action

User selects a quiz radio button. State is held in selectedAnswer. Clicking Execute locks the UI via isSubmitted.

const submitChallenge = () => {
  if (selectedAnswer !== null) {
    setIsSubmitted(true);
  }
};
4

Card Advancement

Loop

Clicking "Advance" iterates cardIndex, resets panelMode to main, and clears the challenge data for the next topic.

setCardIndex(cardIndex + 1);
setPanelMode('main');
setSelectedAnswer(null);
setIsSubmitted(false);
